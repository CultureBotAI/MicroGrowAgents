## How to Generate/Update Tables

### Generate All Tables

```bash
# Generate comprehensive report with all 10 tables
uv run python scripts/reporting/generate_all_tables.py

# Output: notes/COMPREHENSIVE_TABLES_REPORT.md
```

### Generate Individual Tables

The `generate_all_tables.py` script contains individual functions for each table:

1. `generate_organism_statistics_table()` - Table 1
2. `generate_coverage_statistics_table()` - Table 2
3. `generate_confidence_distribution_table()` - Table 3
4. `generate_property_extraction_status_table()` - Table 4
5. `generate_doi_citation_coverage_table()` - Table 5
6. `generate_evidence_snippet_quality_table()` - Table 6
7. `generate_taxonomy_database_coverage_table()` - Table 7
8. `generate_organism_diversity_table()` - Table 8
9. `generate_file_source_distribution_table()` - Table 9
10. `generate_ingredient_processing_status_table()` - Table 10

### Update Existing Documentation

To update tables in existing documentation files:

1. **Run table generation:**
   ```bash
   uv run python scripts/reporting/generate_all_tables.py
   ```

2. **Copy relevant tables to documentation:**
   - From `notes/COMPREHENSIVE_TABLES_REPORT.md`
   - To target documentation files

3. **Key files to update:**
   - `notes/NCBI_INTEGRATION_RESULTS.md` - Tables 1, 2, 3, 7, 8
   - `notes/FINAL_TAXONOMY_INTEGRATION.md` - Table 7
   - `notes/TAXONOMY_VALIDATION_RESULTS.md` - Tables 1, 2, 3
   - `notes/SESSION_SUMMARY_2026-01-08.md` - Summary tables

## Best Practices

1. **Always regenerate after extraction:** Run `generate_all_tables.py` after completing evidence extraction
2. **Version control:** Keep dated copies of tables in session summary documents
3. **Consistency:** Use generated tables rather than manually creating them
4. **Documentation:** Update this index when adding new table types
5. **Validation:** Cross-check table data against raw CSV for accuracy
